## Getting Started

1. Fork the repository to your own GitHub account.
2. Clone your forked repository:
    ```bash
    git clone https://github.com/ossd-s26/more-tab-colors.git
3. Navigate into the project directory:
    ```bash
    cd more-tab-colors

## Making Changes 
1. Create a new branch for your changes:
    ```bash
    git checkout -b feature/your-feature-name
2. Make your changes locally.
3. Commit your changes with a clear yet descriptive commit message:
    ```bash
    git commit -m "Add feature: short description"
4. Push your branch to your forked repository:
    ```bash
    git push origin feature/your-feature-name

## Opening a Pull Request
1. Navigate to the original more-tab-colors repository on GitHub.
2. Open a Pull Request from your branch.
3. Include a brief description explaining what changes were made and why they were necessary.
4. Double check that your contribution follows the project structure and is easy to read and review.

## Best Practices for Contributing
- Prioritize clarity and readability in your code.
- Choose descriptive names for variables and functions.
- Limit each pull request to a single fix or feature.
- Update relevant documentation if your contribution changes how the project works.
- Offer constructive and respectful feedback during reviews.
